# Техническое задание: RPG "Disco"

## 1. Общее описание

Однопользовательская ролевая игра с видом сверху. Основной фокус — диалоги с NPC, активные и пассивные
проверки навыков и развитие персонажа. Управление — мышь (point-and-click). Движок — Pygame.

**Платформы:** Windows (основная), Linux. Сборка через PyInstaller.  

---

## 2. Текущее состояние (реализовано)

### 2.1 Персонаж

1) Передвижение: Point-and-click по тайловой карте, поиск пути через BFS. Анимация ходьбы (4 направления,
10 кадров каждое).
2) Система навыков: 5 основных атрибутов (СИЛА, ЛОВКОСТЬ, ИНТЕЛЛЕКТ, ПСИХИКА, ВОСПРИЯТИЕ), каждый имеет 5
навыков (25 навыков всего).
3) Очки навыков: При повышении уровня начисляется 1 очко. Вложение в атрибут повышает все 5 привязанных
навыков на 1. Есть откат до подтверждения. 
4) Окно навыков: Полноэкранное меню (TAB). Отображает иконки 25 навыков, превью с описанием, кнопки +/-
для атрибутов, счётчик оставшихся очков. 
5) Сохранение: JSON-файл `player.json` хранит: уровень, очки навыков, атрибуты, навыки, позицию, локацию и
флаги состояния(что игрок сделал в мире).
6) Флаги: Словарь произвольных булевых флагов (`player.flags`). Ставятся по ходу диалогов, проверяются при
открытии диалога и при формировании модификаторов проверки. 

### 2.2 Карта и навигация

1) Тайловая карта: 2D-массив: 0 — проходимая клетка, 1 — стена, 2 — интерактивный NPC, 3 — спецтайл.
2) Камера следует за игроком с центрированием. 
3) Локации: Каждая локация — отдельный Python-файл (`src/locations/`), задаёт сетку и привязку NPC к
позициям и диалогам. Сейчас одна тестовая комната с 2 NPC. 

### 2.3 Диалоговая система

1) Структура: Диалог — JSON-файл с узлами. Каждый узел содержит текст NPC и массив вариантов ответа.
Вариант ведёт к другому узлу (`next`), закрывает диалог (`null`) или запускает проверку навыка (`check`). 
2) Условный вход: Массив `entry` в JSON: список правил `{node, flag}`. Правила проверяются сверху вниз,
первое с истинным флагом (или без условия) определяет стартовый узел. Сделано для ситуаций с повторным 
обращением к NPC после определённых событий, например: первое обращение к торговцу -> "Привет, раньше не
видел тебя здесь", повторное обращение -> "А я тебя помню, заходи".
3) Установка флагов: Поле `set_flag` в узле — при входе в узел автоматически ставит указанный флаг игроку.
4) Отрисовка: Окно диалога внизу экрана посередине. Слева — текст NPC с переносом строк. Справа — кнопки вариантов ответа с подсветкой при наведении. 

### 2.4 Проверки навыков

1) Запуск: Вариант ответа с полем `check` запускает экран проверки. 
2) Экран проверки: Экран показывающий класс сложности проверки, d20 кость посередине и текущие модификаторы
проверки(значение соответствующего навыка, определённые условия)
3) Результат: Значения модификаторов складываются с выпавшим значением на d20 и если данное значение будет 
выше класса сложности то диалог пойдёт по ветке "Успех" иначе "Провал"

## 3. Планируемый функционал

### 3.1 Пассивные проверки навыков

**Приоритет:** высокий

Пассивные проверки происходят автоматически при входе в определённый узел диалога — без участия игрока и без экрана с кубиком.

- При входе в узел значение навыка игрока сравнивается с DC.
- При успехе — в текст NPC добавляется дополнительная реплика от имени навыка (отображается другим цветом/стилем, как "голос" навыка).
- При провале — ничего не появляется (или появляется альтернативная реплика, если указана).
- Пассивная проверка не прерывает диалог и не показывает кубик.
- Также может проявляться вне диалога как пассивная проверка на внимательность укажет на ловушку перед 
игроком

**Отображение:**
- Текст навыка выводится отдельным блоком текста перед основным текстом NPC.
- Формат: `[ЭМПАТИЯ — Успех] Он врёт. У него дрожат руки.`

### 3.2 Инвентарь

**Приоритет:** высокий

Система предметов, которые игрок может находить, получать через диалоги и использовать.

**Структура данных:**
- В `player.json` добавляется поле `"inventory": []` — массив ID предметов.
- Предметы описываются в отдельном файле `items.json`:
  ```json
  {
    "pass": {
      "name": "Пропуск",
      "description": "Официальный документ позволяющий пройти в запретные места.",
      "icon": "pass.png",
      "on_pickup_flag": "has_pass"
    }
  }
  ```
- Поле `on_pickup_flag` — при получении предмета автоматически ставится флаг (интеграция с существующей
системой флагов и модификаторов проверок).

**UI:**
- Окно инвентаря открывается отдельной клавишей (I) или переключением на вторую вкладку из окна навыков
- Сетка иконок предметов с превью (название + описание) при наведении.
- В диалогах — возможность получать и отдавать предметы через поле в JSON-узле:
  ```json
  "give_item": "pass",
  "take_item": "old_key"
  ```

### 3.3 Здоровье

**Приоритет:** высокий

**Механика:**
- В `player.json` добавляются поля `"hp": 10, "max_hp": 10`.
- HP теряется от определённых событий: провал опасной проверки, сюжетный урон.
- HP восстанавливается через предметы или сюжетные события.
- При HP = 0 — экран смерти с возможностью загрузить сохранение.

**Отображение:**
- Круг HP в углу экрана с числовым значением (видна всегда, кроме меню навыков).

### 3.4 Шкала опыта

**Приоритет:** высокий

**Механика:**
- В `player.json` добавляются поля `"xp": 0, "xp_to_next": 100`.
- Опыт начисляется за: завершение диалогов, успешные проверки навыков, выполнение квестов.
- При достижении порога — автоматический `level_up()` (сейчас реализован ручной вариант, новый уровень по 
нажатии на E), порог увеличивается.
- Источники XP задаются в JSON: поле `"xp_reward"` в узлах диалога и квестах.

**Отображение:**
- Полоска опыта рядом с HP.

### 3.5 Квесты

**Приоритет:** средний

Система отслеживания задач игрока.

**Структура данных:**
- Квесты описываются в `quests.json`:
  ```json
  {
    "find_pass": {
      "title": "Найти пропуск",
      "description": "Получить пропуск, чтобы пройти мимо стражника.",
      "stages": [
        {"text": "Поговорить со стражником", "complete_flag": "met_guard"},
        {"text": "Получить пропуск у начальника", "complete_flag": "has_pass"},
        {"text": "Вернуться к стражнику", "complete_flag": "guard_passed"}
      ],
      "xp_reward": 50
    }
  }
  ```
- Активные квесты хранятся в `player.json`: `"active_quests": ["find_pass"]`, `"completed_quests": []`.
- Стадии квеста отслеживаются автоматически через существующие флаги — каждая стадия привязана к флагу.
- Квест выдаётся через поле в узле диалога: `"start_quest": "find_pass"`.
- При выполнении всех стадий — квест автоматически перемещается в `completed_quests`, начисляется XP.

**UI:**
- Журнал квестов открывается клавишей (J) или переключением на третью вкладку из окна навыков.
- Список активных квестов с текущей стадией. Выполненные стадии — зачёркнуты или отмечены.

### 3.6 Несколько комнат

**Приоритет:** средний

**Механика:**
- Переходы между локациями задаются через тайл типа перехода (новый тип, например `4`) на карте.
- В описании локации указывается, куда ведёт переход:
  ```python
  transitions = {
      (15, 5): {"location": "town_square", "spawn": (0, 5)}
  }
  ```
- При наступании на тайл перехода — загрузка новой локации, перемещение игрока в точку `spawn`.
- Текущая локация сохраняется в `player.json` (поле `location` уже реализовано).

**Загрузка:**
- Локации загружаются динамически по строковому ID через `importlib`.
- Каждая локация — отдельный файл в `src/locations/`.

### 3.7 Настройки

**Приоритет:** низкий

**Параметры:**
- Громкость музыки (0–100).
- Громкость звуковых эффектов (0–100).
- Разрешение экрана / полноэкранный режим.

**Хранение:**
- Файл `settings.json` в корне проекта.
- Загружается при старте, применяется к `pygame.display` и `pygame.mixer`.

**UI:**
- Экран настроек доступен из главного меню (ESC или стартовый экран).

### 3.8 Звук

**Приоритет:** низкий

**Компоненты:**
- Фоновая музыка — отдельный трек для каждой локации. Зацикленное воспроизведение через `pygame.mixer.music`.
- Звуковые эффекты: клик по кнопке, бросок кубика, успех/провал проверки, шаги, открытие/закрытие диалога.
- Звуки хранятся в `assets/sounds/`.
- Озвученные реплики в диалогах


### 3.9 Пошаговая боевая система

**Приоритет:** очень низкий (если хватит времени)

**Механика:**
- Бой запускается из диалога (аналогично проверке навыка) или при входе на определённый тайл.
- Пошаговый режим: игрок и противник ходят по очереди.
- Атака — бросок d20 + модификатор навыка "Ближний бой" или "Меткость" против класса защиты противника.
- Урон зависит от навыков и экипировки.
- У противника своя шкала HP.
- Победа/поражение ведёт к соответствующему узлу диалога или экрану смерти.

**Данные противников:**
- Описываются в JSON:
  ```json
  {
    "guard_fight": {
      "name": "Стражник",
      "hp": 15,
      "ac": 12,
      "attack_bonus": 3,
      "damage": "1d6+1"
    }
  }
  ```

**UI:**
- Отдельный экран боя с отображением HP обоих участников, кнопками действий (атака, предмет, побег) и анимацией бросков.

## 5. Форматы данных

### 5.1 Узел диалога (JSON)

```json
{
  "node_id": {
    "text": "Текст NPC",
    "set_flag": "flag_name",
    "give_item": "item_id",
    "take_item": "item_id",
    "xp_reward": 25,
    "start_quest": "quest_id",
    "passive_check": {
      "skill": "skill_name",
      "dc": 10,
      "success_insert": "Текст голоса навыка при успехе",
      "failure_insert": null
    },
    "options": [
      {"text": "Текст варианта", "next": "node_id"},
      {"text": "Текст варианта", "next": null},
      {
        "text": "[НАВЫК] Текст варианта",
        "check": {
          "skill": "skill_name",
          "dc": 12,
          "modifiers": [
            {"label": "Описание", "value": 2, "flag": "flag_name"}
          ],
          "success": "success_node_id",
          "failure": "failure_node_id"
        }
      }
    ]
  }
}
```

Все поля кроме `text` и `options` — опциональны. Добавление новых полей не ломает существующие диалоги.

### 5.2 Сохранение игрока (player.json)

```json
{
  "level": 1,
  "xp": 0,
  "xp_to_next": 100,
  "skill_points": 0,
  "hp": 10,
  "max_hp": 10,
  "attributes": [1, 1, 1, 1, 1],
  "skills": {"fortitude": 1, "...": 1},
  "position": {"grid_x": 1, "grid_y": 1},
  "location": "test",
  "inventory": [],
  "active_quests": [],
  "completed_quests": [],
  "flags": {}
}
```
